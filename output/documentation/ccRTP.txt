<h3 id="introduction">Introduction to GNU RTP Stack</h3>

<p>GNU ccRTP is an implementation of RTP, the real-time transport protocol
from the <a href="https://www.ietf.org/">IETF</a> (see <a
href="https://www.ietf.org/rfc/rfc3550.txt">RFC&nbsp; 3550</a>, <a
href="https://www.ietf.org/rfc/rfc3551.txt">RFC&nbsp;3551</a> and  <a
href="https://www.ietf.org/rfc/rfc3555.txt">RFC&nbsp;3555</a>). Releases
1.5.0 and above support the Secure RTP profile, SRTP (see <a
href="https://www.ietf.org/rfc/rfc3711.txt">RFC&nbsp;3711</a>). A separate
extension package, <a href="/software/ccrtp/zrtp.html">GNU ZRTP</a>
(libzrtpcpp), provides support for the ZRTP protocol (as defined in the
Internet <a href="https://tools.ietf.org/html/rfc6189">RFC&nbsp;6189</a>).</p>

<p>ccRTP is a C++ library based on <a href="/software/commoncpp/commoncpp.html">
GNU Common&nbsp;C++</a>, which provides a high performance, flexible and
extensible standards-compliant RTP stack with full RTCP support. The design
and implementation of ccRTP make it suitable for high capacity servers and
gateways as well as personal client applications.</p>

<p>In designing ccRTP, we have taken into account that RTP has been defined
as an application-level protocol framework rather than a typical Internet
transport protocol such as TCP and UDP. Thus, RTP is hardly ever implemented
as a layer separated from the application. Consequently, RTP applications
often must customize the adaptable RTP packet layout and processing rules,
timing constraints, session membership rules as well as other RTP and RTCP
mechanisms. ccRTP aims to provide a framework for the RTP framework, rather
than being just an RTP packet manipulation library.</p>

<p>Support for both audio and video data is also considered in the design of
ccRTP, that can do partial frame splits/re-assembly. Unicast, multi-unicast
and multicast transport models are supported, as well as multiple active
synchronization sources, multiple RTP sessions (SSRC spaces), and multiple
RTP applications (CNAME spaces). This allows its use for building all forms
of Internet standard-based audio and visual conferencing systems.</p>

<p>GNU ccRTP is threadsafe and high performance. It uses packet-queue lists
for the reception and transmission of data packets. Both inter-media and
intra-media synchronization is automatically handled within the incoming
and outgoing packet queues. GNU ccRTP offers support for RTCP and many other
standard and extended features that are needed for both compatible and
advanced streaming applications. It can mix multiple payload types in
stream, and hence can be used to implement <a
href="https://www.ietf.org/rfc/rfc2833.txt">RFC&nbsp;2833</a>-compliant
signaling applications as well as other specialized things. GNU ccRTP also
offers direct RTP and RTCP packet filtering.</p>

<p>GNU ccRTP uses templates to isolate threading and sockets-related
dependencies, so that it can be used to implement realtime streaming with
different threading models and underlying transport protocols, not just
with IPV4 UDP sockets. For a more detailed list of ccRTP features, you can
have a look at the programmers' manual.</p>

<h3 id="development">Development</h3>

<p>[2017] &ndash; The latest release is GNU ccRTP 2.1.2.</p>

<p>[2007] &ndash; Release 1.5.0 introduces support for the Secure RTP profile,
SRTP (<a href="https://www.ietf.org/rfc/rfc3711.txt">RFC&nbsp;3711</a>), which
allows for secure connections, and voice encryption using either libgcrypt or
openSSL backends.  In addition, native support for Phil Zimmermann's ZRTP
protocol, which offers a transparent way to transfer encrypted voice over
existing infrastructure, is released as a separate extension library
licensed under the GNU GPL. This work was completed with the incredible
effort of Werner Dittmann, and extensive help by Michel de Boer, who works
on the Twinkle Softphone.</p>

<p>[2006] &ndash; The current 1.4.x releases have focused on introducing IPV6
support, simplifying use, and improving performance. There is some limited use
of STL in ccRTP for managing destination lists, and to improve embedded
profiles, as well as to better support the broken state of STL when used in
DLL's on some platforms; this will likely be removed in future releases.
Adding generic APP headers for RTCP is another area for new releases to focus
on.</p>

<h3 id="downloading">Downloading</h3>

<p>Releases 0.6.1 and above are found on the <a
href="https://ftp.gnu.org/gnu/ccrtp/">GNU FTP site</a> or one of its <a
href="/prep/ftp.html">mirrors</a>, along with the extension package
libzrtpcpp. Please use a <a
href="https://ftpmirror.gnu.org/ccrtp/">nearby mirror</a> if you can.</p>

<p>GNU ccRTP may be available from other sites in various package formats,
including GNU/Linux RPM and Debian packages, <code>.pkg</code> files under
various Unix operating systems, etc. I do encourage people to distribute GNU
ccRTP in a package form where convenient for installation, so long as
source is also made available.</p>

<h3 id="documentation">Documentation</h3>

<p>A programmers' manual and an exhaustive reference manual are distributed
with the source packages of GNU ccRTP. The first is written in GNU Texinfo,
whereas the second is automatically generated from the sources through
Doxygen. They are available online for several releases of ccRTP:</p>

<ul>
  <li><a href="doc/manual/html/">ccRTP Programmers' Manual</a>
  (release&nbsp;1.3);</li>

  <li><a href="doc/refman/html/">ccRTP Reference Manual</a>
  (release&nbsp;2.1.2), also documenting libzrtpcpp-2.3.4;</li>

  <li>class documentation for ccRTP 1.7.2, as part of the larger <a 
  href="/software/commoncpp/docs/refman/html/">Common&nbsp;C++&nbsp;2 /
  Bayonne&nbsp;2 framework</a>.</li>
</ul>

<h3 id="participating">Participating</h3>

<p>As the current GNU ccRTP maintainer, I am very open to bringing new ideas
and code into GNU ccRTP. In particular, I am looking for help from
contributors who have a deep expertise in Internet Audio and Visual
conferencing. The source code repositories can be accessed anonymously
through the <a href="https://sv.gnu.org/projects/ccrtp/">GNU ccRTP
project</a> on Savannah; I would be happy to receive and evaluate patches for
inclusion in future releases.</p>

<p>The ccRTP mailing list is <a
href="mailto:ccrtp-devel@gnu.org">&lt;ccrtp-devel@gnu.org&gt;</a>. It
has a <a href="https://lists.gnu.org/mailman/listinfo/ccrtp-devel">web
interface</a> for subscribing, modifying subscription options and searching
the list archives. Feel free to send any bug reports, patches, and questions
to this list.</p>
