<h2>GNU Shishi</h2>

<h3><a href="#TOCintroduction" id="introduction">Introduction</a></h3>

<p>
  This page contains information about Shishi, a free implementation of
  the Kerberos 5 network security system.
</p>

<p>
  If you do not know what Kerberos 5 is, I suggest to read
  the <a href="https://www.ietf.org/rfc/rfc4120.txt">Kerberos V5
    standard</a>.  Also see the <a href="research.html">page with
    related research papers</a> that may be of interest.
</p>

<p>
  The goals of this project are:
</p>

<ul>
  <li>Full standards compliance.</li>
  <li>Thread safe library.</li>
  <li>Internationalization, both for client messages but also for
    non-ASCII username and passwords.</li>
  <li>Integrate with existing password management systems
    (/etc/passwd, PAM, SASL).</li>
  <li>Support authentication using OpenPGP and X.509 through
    TLS, including smart card support.</li>
  <li>Clean room implementation with clear copyright and
    license.</li>
</ul>

<p>
  Shishi is licensed under the GPLv3, and the Shishi manual is
  licensed under the GFDL.
</p>

<p>
  <a class="FlattrButton" style="display:none;"
     href="/software/shishi/"></a>
</p>

<h3><a href="#TOCstatus" id="status">Documentation and Status</a></h3>

<p>
  Refer to the <a href="manual/">Shishi Manual</a> web page for links
  to the manual in all formats; however, quick links to the most
  popular formats:
</p>

<ul>
  <li><a href="manual/shishi.html">Online HTML manual</a></li>
  <li><a href="manual/shishi.pdf">PDF manual</a></li>
  <li><a href="reference/">API reference in HTML</a></li>
</ul>

<p>
  Shishi has received some real-world testing and should be considered
  stable, although it is still a fairly young implementation.  Basic
  support for acquiring and managing tickets are working, as well as
  serving requests in a Key Distribution Center daemon.  DES, 3DES and
  AES cipher suites are supported. A <i>PAM</i> module for host
  security is included, as well as a Shishi port of a
  <i>rsh</i>/<i>rlogin</i> client.
</p>

<p>
  A <i>telnet</i> client and server with Kerberos authentication is
  supported via <a href="/software/inetutils/">GNU InetUtils</a>.
  A <i>SSH</i> client and server with Kerberos authentication is
  supported via <a href="/software/gss/">GSS</a>
  and <a href="/software/gss/gss-lsh.html">GSS-LSH</a>.  A <i>IMAP</i> server
  with Kerberos authentication (GSSAPI SASL mechanism) is supported
  via <a href="https://mailutils.org">GNU MailUtils</a>.  A <i>IMAP</i>
  command line client with Kerberos authentication (GSSAPI SASL
  mechanism) is supported via <a href="/software/gsasl/">GNU SASL</a>, which
  also provide authentication (including Kerberos 5 via GSSAPI) via
  the SASL API for any application.
</p>

<p>
  Shishi is developed for the GNU/Linux system, but runs on over 20
  platforms including most major Unix platforms and Windows, and many
  kind of devices including iPAQ handhelds and S/390 mainframes.
</p>

<p>
  Shishi requires <a href="/software/libtasn1/">GNU Libtasn1</a>, which is
  included in the package, so you do not need to install it
  separately.
</p>

<p>
  Shishi can optionally use <a href="https://gnutls.org/">GnuTLS</a> (for
  OpenPGP and X.509 authentication), <a href="/software/libidn/">GNU
  Libidn</a> (recommended for non-ASCII support),
  and <a href="https://www.gnupg.org/related_software/libgcrypt/">GNU
  libgcrypt</a>.
</p>

<h3><a href="TOCnews" id="news">News</a></h3>

<p>
  Note that new releases are only mentioned here if they introduce a
  major feature or is significant in some other way.  Read
  the <a href="https://lists.gnu.org/archive/html/info-gnu/">info-gnu</a>
  mailing list if you seek more frequent announcements.
</p>

<ul>
  <li>2010-05-20: <a href="https://lists.gnu.org/archive/html/help-shishi/2010-05/msg00000.html">
      Version 1.0.0</a> released, takes Shishi out of alpha testing.</li>

  <li>2007-06-29: <a href="https://web.archive.org/web/20091215053203/https://article.gmane.org/gmane.comp.gnu.shishi.general/425">
      Version 0.0.32</a> released under the GPLv3.</li>

  <li>2006-03-25: <a href="https://web.archive.org/web/20091215053157/http://article.gmane.org/gmane.comp.gnu.shishi.general/24">
      Version 0.0.23</a> released, mainly as a foundation to build
      official Debian packages.</li>

  <li>2006-01-17: Experimental Shishi packages for Debian are
    available.</li>

  <li>2004-11-12: The experimental STARTTLS support is now
    <a href="https://josefsson.org/krb5starttls/">documented in an
      Internet draft</a>.</li>

  <li>2004-06-13: The InetUtils work has <a href="feg-inetutils/">its
      own home page</a> now, and will be a playground for new features
    (even non-Shishi related) in GNU InetUtils.</li>

  <li>2004-01-22: New releases are no longer announced here, and
    hasn't been for a while.  Instead,
    read <a href="https://lists.gnu.org/archive/html/info-gnu/">info-gnu</a>
    or check the release directory from time to time.  By the
    way, <a href="https://alpha.gnu.org/pub/gnu/shishi/">Shishi
      0.0.14</a> was just released.</li>

  <li>2004-01-13: A new snapshot of GNU InetUtils with Shishi support
    was released.  Build it as usual for Kerberos support, i.e.,
    with <code>--enable-encryption --enable-authentication</code>.  It
    includes telnet(d), rsh(d) and rlogin(d) with Shishi support.</li>

  <li>2004-01-01: Savannah had problems last month, and still isn't
    operating fully.  CVS has been moved to a private machine, a
    read-only mirror of it will hopefully be available via Savannah in
    the future.</li>

  <li>2003-10-16: Shishi 0.0.8 released.  STARTTLS upgrade of TCP
      connections (only anonymous DH for now).  Password processing
      via SASLprep instead of KRBprep.  Authorization and
      documentation improvements.</li>

  <li>2003-10-11: Snapshot of Shishi-port of GNU InetUtils released,
    contains telnet(d) and rsh(d) with Kerberos 5 support via Shishi.
    Developed by Nicolas Pouvesle.</li>

  <li>2003-10-01: Shishi-port of GNOME Ticket Applet added,
    see <a href="#screenshot">below</a>.</li>

  <li>2003-09-21: Shishi 0.0.7 released.  DES-CBC-CRC and ARCFOUR
      works.  Short-hand aliases for encryption type names are
      supported.</li>

  <li>2003-09-14: Shishi 0.0.6 released.  Proxiable, proxy,
    forwardable and forwarded tickets supported.  Man pages for all
    public functions are included. The internal crypto interface now
    fully modularized.</li>

  <li>2003-09-07: Shishi 0.0.5 released.  SAFE and PRIV fixes.  Server
      name to realm mapping via DNS.  Reference manual.</li>

  <li>2003-08-31: Shishi 0.0.4 released.  KDC works.  Shishi port of
      rsh/rlogin client, contributed by Nicolas Pouvesle, included.
      Accompanies <a href="/software/gss/"> GSSLib 0.0.5</a>.</li>

  <li>2003-08-25: Shishi becomes a GNU project.</li>

  <li>2003-08-22: Shishi 0.0.3 released.</li>

  <li>2003-08-17: Shishi 0.0.2 released.</li>

  <li>2003-08-10: Shishi 0.0.1 released.  Few new features, but
    improved internally.  Accompanies <a href="/software/gss/"> GSSLib
    0.0.4</a>.</li>

  <li>2003-06-23: A patch for telnet(d) in <a href="/software/inetutils/">GNU
      InetUtils</a> that implement Kerberos 5 authentication via
    Shishi <a href="https://lists.gnu.org/archive/html/bug-inetutils/2003-06/msg00044.html">
      is published</a>.</li>

  <li>2003-06-02: Shishi 0.0.0 released.  No major changes compared to
    last snapshot, but used by <a href="../gss/">Generic Security
    Services API (GSS-API) 0.0.0</a>.</li>

  <li>2003-02-11: Another snapshot release.  Used
      by <a href="/software/gsasl/">GNU SASL</a>.</li>

  <li>2002-12-26:
      Moved <a href="https://savannah.nongnu.org/projects/shishi/">
      project to savannah</a>.</li>

  <li>2002-12-14: Second snapshot released.</li>

  <li>2002-12-13: PAM works.</li>

  <li>2002-12-08: Telnetd works.</li>

  <li>2002-12-01: Web page opened and a snapshot released.</li>

  <li>2002-11-25: Telnet works.</li>

  <li>2002-10-26: Code moved into CVS.</li>

  <li>2002-09-30: Started coding.</li>
</ul>

<h3><a href="#TOCsupport" id="support">Support</a></h3>

  you can reach it by sending e-mail
  to <a href="mailto:help-shishi@gnu.org">help-shishi@gnu.org</a>.
  Archives of the mailing list discussions, and an interface to manage
  subscriptions, is available through the World Wide Web at
  <a href="https://lists.gnu.org/mailman/listinfo/help-shishi">
    http://lists.gnu.org/mailman/listinfo/help-shishi</a>.
</p>
